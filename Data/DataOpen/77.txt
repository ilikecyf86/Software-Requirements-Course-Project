Allow explicitly set JAVA_OPTS for Workspace-agent and Maven-server
kind/task team/platform sprint/current-sprint status/in-progress 
Description It would be nice if we could explicitly configure JAVA_OPTS for Workspace-agent and Maven-server in such way. If user want's to configure it himself he could set some environment variables. Admin can set defaults to this values. Propose variables CHE_WORKSPACE_AGENT_JAVA_OPTS - user-defined value if JAVA_OPTS for workspace agent. CHE_WORKSPACE_AGENT_JAVA_OPTS_DEFAULT - admin-defined defaults for JAVA_OPTS of workspace agent. Used if CHE_WORKSPACE_AGENT_JAVA_OPTS is not set CHE_WORKSPACE_MAVEN_SERVER_JAVA_OPTS - user-defined value if JAVA_OPTS for maven server. Used if CHE_WORKSPACE_AGENT_JAVA_OPTS_DEFAULT is not set CHE_WORKSPACE_MAVEN_SERVER_JAVA_OPTS_DEFAULT - admin-defined value if JAVA_OPTS for maven server. Used if CHE_WORKSPACE_MAVEN_SERVER_JAVA_OPTS is not set Related to Xmx value for ws-agent's JAVA_OPTS must not exceed total machine RAM #4152 [che-worspaces] Tune OpenJDK JVM flags so as to use less memory fabric8-services/fabric8-tenant-che#79 Tuning workspaces JAVA_OPTIONS redhat-developer/rh-che#451
