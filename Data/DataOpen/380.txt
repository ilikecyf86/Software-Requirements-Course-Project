Hash "#" character in filename causes multiple issues
kind/bug severity/P2 team/plugin status/open-for-dev 
Steps to reproduce: Create a file with # in its name, say, new#new The file gets successfully created with the abovementioned name (API returns 201) However, when opening a file, GET request fails with 404, since Che looks for a file named new, ie everything after # is cut out. Therefore, it is impossible to open, edit or delete a file. Perhaps, file name should be validated not to contain #
