Cursor is set up to the next line after using autocomplete feature in the C# files
kind/bug severity/P2 
Reproduction Steps: Run C# stack with default project Open any C# file until C# languageserver will be started Delete the part of any method and use "Ctrl + Space" Pay attention to the cursor location Expected behavior: The cursor is located at the end of the row where autocomplete was launched Observed behavior: The cursor is located at the next row OS and version: Che 5.18.0 ; Chrome 59.0 ; Fedora 25
