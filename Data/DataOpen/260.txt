[OpenShift]: Workspace Name created via Factory does not confirm Kubernetes naming
kind/bug team/osio 
Workspace Name when creating che workspaces using Factory does not adhere to Kubernetes Naming pattern, which is causing the creation of workspaces via factory fails when there exists a workspace with same name. Reproduction Steps: create a workspace using factory from url https://github.com/kameshsampath/che-boot-demo, the will create workspace called che/kameshsampath Now create one more workspace again using factory from url https://github.com/kameshsampath/che-boot-demo, the will create a new namespace with name che/kameshsampath_1 The name che/kameshsampath_1 does not confine to Kubernetes Name pattern hence will throw the following error: Unable to start workspace agent.
