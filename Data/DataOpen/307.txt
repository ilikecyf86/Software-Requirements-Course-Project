Autocomplete feature for macros in the commands editor does not recognized the text before the character "/"
kind/bug severity/P2 
Reproduction Steps: Run java stack with default spring project Go to the commands explorer Click twice on the "build" command Set cursor to the "Preview URL" text field Type "80/tcp" and press "Ctrl + Space" on the keyboard In the opened window pay attention that only "tcp" text is highlighted Click on any macros Pay attention that macros is added after "80/" text Expected behavior: Autocomplete feature will be recognized all text and all text will be changed by macros Observed behavior: Text before "/" is not recognized OS and version: CHE 5.18.0 ; Chrome 59.0 ; Fedora 25
