Sometimes commited changes from previous branch appears after checkout to another
kind/bug status/open-for-dev team/enterprise status/in-progress status/open-for-dev sprint/current-sprint status/code-review status/in-progress status/code-review sprint/current-sprint 
In case of checkout to some branch and quick checkout back changes from the second branch is added into the first branch even if no changes was made by a user. Note, that both branches were 'clean'. Reproduction Steps: Create a workspace with a project. Create an additional branch with some changes. Commit changes. Open terminal. Checkout to master, check git status that nothing to commit, working tree clean. Checkout to the additional branch and then checkout to master again immediately. Repeat step 4 a few times. Observed behaviour: Some files are changed after checkout! Expected behaviour: No changes after checkout. OS and version: Ubuntu 16.04
