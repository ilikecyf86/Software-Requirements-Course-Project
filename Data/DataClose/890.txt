Undo does not autosave the editor
kind/bug severity/P1 team/ide sprint/next-sprint sprint/next-sprint status/in-progress status/code-review status/pending-merge status/in-progress status/code-review status/pending-merge 
It seems that undo (Ctrl+Z) does not trigger the autosave of the editor. This may leave broken files without the user noticing it. Reproduction Steps: Open a file in the editor Delete a character Wait 5 seconds to ensure that the autosave will save the changes to the file Hit Ctrl+Z to undo the change - the deleted character is now back in the editor Wait 5 seconds to give a chance to the autosave Close the editor Open again the saved file in the editor PROBLEM: The deleted character (step 2) that was undone (step 4) is still missing.
