Remove temporary workspaces on server shutdown
kind/task status/analyzing team/platform sprint/next-sprint status/analyzing sprint/next-sprint 
Temporary workspaces are removed only when workspace stop method is called directly, so if che server is stopped and temporary workspaces are running they won't be removed, unless you start and stop them again. Reproduction Steps: Start a temporary workspace Stop che server in any possible way e.g. che.sh stop Start che server Expected behavior: Temporary workspaces removed Observed behavior: Temporary workspace are listed with STOPPED status Additional information: Problem can be reliably reproduced, doesn't happen randomly: YES The simplest way to fix this issue is to remove temporary workspace on WorkspaceStatus[STOPPED] event.
