After switching between two workspaces, preference file is not updated
kind/bug severity/P2 team/ide status/in-progress status/code-review status/in-progress status/code-review 
After switching between two workspaces, preference file is not updated and workspaces are restored on the snapshotted point. Reproduction Steps: Create two java stack workspaces In the first workspace open several files and stopping workspace with snapshot Run the first workspace Close restored editor tabs and open several new files Switch to the second workspace, after that return to the first workspace Pay attention that workspace will be restored on the snapshotted point. Expected behavior: Workspace will be restored with new opened tabs Observed behavior: Workspace will be restored with tabs that were last snapshotted OS and version: CHE 5.16.0 ; Chrome 59.0 ; Fedora 25 Attachment:
