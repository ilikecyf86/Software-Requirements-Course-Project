Git History panel is not synchronized with projects in project tree
kind/bug 
Selecting other project under Git in the project tree does not affect on 'Git History' panel Reproduction Steps: Prepare more than one projects under Git Select any project under git and click menu 'Git' -> 'Show History...' Select other project under git and click menu 'Git' -> 'Show History...' Expected behavior: 'Git History' panel must refresh its data according to selected project Observed behavior: Commits of first selected project are showed in 'Git History' panel Che version: 5.0.0-M8-SNAPSHOT OS and version: Ububtu 16.04 Docker version: 1.12.1 Che install: Docker container Additional information: When removing git from project or just deleting project, the 'Git History' panel called from that project will be still present.
