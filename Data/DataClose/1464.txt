Wrong behavior of create and delete java files with invalid names
kind/bug sprint/next-sprint team/ide sprint/current-sprint sprint/next-sprint 
User can create java classes with invalid names , besides error message is not shown . If try to delete this classes error message will be shown or classes will be deleted , but after refresh page this classes will be restored in project explorer tree . Reproduction Steps: Open a project in the IDE (e.g. web-java-spring from template) Try to create java class with invalid name : Pay attention to the project tree Try to delete this java class Pay attention to the screen Expected behavior : User not able to create the java classes with invalid names , and error message will be shown . After refresh page deleted java classes will not restore in project tree. Observed behavior : User able to create java classes with invalid names , error message not displayed . Java classes with invalid names will be restored in project explorer tree after refresh page .
