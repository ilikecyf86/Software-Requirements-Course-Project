JSONRPC connection doubles messages after start/stop of WS without page refresh
kind/bug team/ide status/analyzing 
When a user starts WS and then stops it and starts again without refreshing browser page JSONRPC connection starts sending multiple copies of the same events, e.g. WS status change events. Reproduction Steps: Open IDE. Open dev tools in the browser. Find WebSocket connection to master and start tracking WebSocket frames. Start WS X. Stop it with the menu. Ensure that there is only 1 copy of WS status messages. Start it again without refreshing browser tab. Ensure that there is several copies of each WS status event appear.
