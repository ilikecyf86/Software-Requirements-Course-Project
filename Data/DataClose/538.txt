Checking out from a branch with a file open that exists only on that specific branch does not close the editor tab that has it opened
kind/question 
Checking out from a branch with a file that exists only on the previous branch makes the IDE hang on "getting item" Loader state and does not close the editor tab with the nonexistent file. Can be reproduced with this Stack Reproduction Steps: 1:) Using the factory I provided, go to project root and do git checkout development 2:) try navigating to lib, open a file, don't close. 3:) run git checkout to release or lts Expected Behavior: the tab with the nonexistent file would close. Observed Behaviour: IDE goes on a long "Getting Item" Loader And checking to dev console, you get this type of errors OS and version: Codenvy 5.17.0-SNAPSHOT on Windows 10
