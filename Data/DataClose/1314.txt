NPE in FileWatcherService
kind/bug team/ide sprint/next-sprint sprint/current-sprint sprint/next-sprint 
In some cases after deleting of project subdirectories NPE in FileWatcherService occurs. Reproduction Steps: Create a workspace from Java stack Clone https://github.com/jboss-developer/jboss-eap-quickstarts.git and configure as a blank project Unfold the project and delete all files and folders using one operation (use shift and left mouse button to select all files faster) Click on dev-machine tab in Processes panel and wait a bit. OS and version: Ubuntu 16.04 Additional info: Problem can be reliable reproduced. CHE version: 5.6.0-SNAPSHOT Logs with NPE: FileWatcherService_NPEs.txt
