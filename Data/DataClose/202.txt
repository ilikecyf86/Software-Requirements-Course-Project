Single-port routing with multi-user mode: Gateway Timeout
kind/question 
Description When enabling single port routing on a multi user instance of Che, loading Che in browser fails. It needs some time and returns the error Gateway Timeout. Single port routing on a single user instance works fine. Reproduction Steps Creating Che conf with docker run -it --rm -v /var/run/docker.sock:/var/run/docker.sock -v /data/che:/data eclipse/che Setting the following parameters in che.env: CHE_SINGLE_PORT=true CHE_MULTIUSER=true Starting Che with docker run -it --rm -v /var/run/docker.sock:/var/run/docker.sock -v /data/che:/data eclipse/che start (--debug) OS and version: Docker stable on Debian stretch amd64, lastest Che version from Docker Hub. I'm running this in my local network, so there's no firewall inbetween. Diagnostics: Log of che_traefik_1 container: time="2017-10-27T17:13:08Z" level=warning msg="Error forwarding to http://172.20.0.2:8080, err: dial tcp 172.20.0.2:8080: i/o timeout"
